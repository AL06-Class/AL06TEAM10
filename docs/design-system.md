# 디자인 시스템

모든 화면이 같은 디자인 기준을 따르도록 정리한 최소 디자인 시스템입니다.

## 목적

- 수강생마다 다른 AI에게 화면을 맡겨도 결과물이 한 프로젝트처럼 보이게 합니다.
- 각 페이지가 임의의 색상, 여백, 버튼, 카드 스타일을 만들지 않게 합니다.
- 새 화면은 이 문서를 기준으로 만들고, 기준이 부족할 때만 문서를 먼저 수정합니다.
- 디자인 시스템은 개발을 막는 사전 승인 장치가 아니라, 화면의 결이 달라졌을 때 다시 맞추는 기준입니다.

## 기본값

### 디자인 방향

- Apple 웹 디자인 참고안을 바탕으로, 조용하고 프리미엄한 업무 도구 UI를 기본으로 합니다.
- 화면의 주인공은 장식이 아니라 검증된 트레이너 프로필, 평가 결과, 비교 데이터입니다.
- 장식보다 정보 구조, 여백, 선명한 사진/데이터, 사용성을 우선합니다.
- 모바일에서도 깨지지 않게 만들되, 데스크톱 작업 화면을 함께 고려합니다.
- 같은 역할의 UI는 같은 모양과 동작을 유지합니다.

### 공통 레이아웃

- 본문 최대 너비는 기존 앱 기준을 따릅니다.
- 역할별 화면은 같은 간격, 카드, 표, 제목 체계를 사용합니다.

### 운영 방식

- 새 화면은 가능한 한 이 문서의 기본값을 참고해서 만듭니다.
- 기준이 부족하면 먼저 기능을 개발하고, 반복되는 패턴만 문서에 추가합니다.
- 이미 만든 화면이 서로 다른 톤으로 보이면 후속 정리 작업으로 맞춥니다.
- 모든 PR에서 디자인 시스템 완전 준수를 강제하지 않습니다.
- 단, 기본 색상, 버튼, 카드, 표 기준은 크게 벗어나지 않습니다.
- 디자인 정리가 필요하면 AI에게 "디자인 시스템 기준으로 화면 톤을 정리해줘"라고 요청합니다.

### 색상

- 메인 컬러는 Apple 참고안의 조용한 액션 블루 계열(`#0066cc`)을 기본으로 합니다.
- 배경은 흰색과 아주 옅은 회색(`#f5f5f7`)을 기본으로 하고, 강조 구간에만 차콜 계열 어두운 면을 사용합니다.
- 텍스트는 순검정 대신 차콜 계열 진한 회색(`#1d1d1f`)을 사용하고, 보조 텍스트는 중간 회색을 사용합니다.
- 성공, 경고, 오류 색상은 의미가 있을 때만 사용합니다.
- 페이지별로 별도 컬러 테마를 만들지 않습니다.
- 장식용 그라데이션이나 두 번째 강조색은 만들지 않습니다.

### 타이포그래피

- 기본 폰트는 시스템 폰트를 사용합니다.
- 제목은 과하게 크지 않게 600 굵기 중심으로 사용하고, 본문은 읽기 쉬운 16~17px 기준을 사용합니다.
- 제목, 섹션 제목, 본문, 보조 텍스트 단계만 사용합니다.
- 과한 대형 제목이나 마케팅 페이지식 문구는 사용하지 않습니다.
- 버튼과 배지 안의 텍스트는 모바일에서도 잘리지 않게 합니다.
- 참고안처럼 촘촘한 인상을 주되, 구현에서는 글자 간격을 음수로 줄이지 않습니다.

### 컴포넌트

- 버튼은 기본, 보조, 위험 버튼으로만 구분합니다.
- 주요 행동 버튼은 파란색 pill 형태를 기본으로 하고, 보조 버튼은 흰색/투명 배경에 얇은 선으로 구분합니다.
- 입력창은 라벨, 안내 문구, 오류 문구 기준을 통일합니다.
- 카드는 반복 목록이나 독립 정보 묶음에만 사용하고, 그림자보다 1px 선과 여백으로 구분합니다.
- 표는 데이터 비교와 관리 화면에 사용합니다.
- 아이콘은 필요할 때만 사용하고, 같은 의미에는 같은 아이콘을 사용합니다.
- 프로필 사진, 인증 배지, 평가 점수처럼 신뢰를 만드는 실제 정보가 UI의 중심이 되게 합니다.

### 금지 기준

- 화면마다 다른 색상 테마를 만들지 않습니다.
- 페이지 전체를 카드처럼 감싸지 않습니다.
- 불필요한 장식용 그래픽이나 배경 효과를 넣지 않습니다.
- 카드, 버튼, 제목에는 과한 그림자를 쓰지 않습니다.
- 사진이나 프로필 이미지는 흐릿한 장식 배경으로 쓰지 않고, 사용자가 실제 정보를 확인할 수 있게 보여 줍니다.

## 논의할 항목

- 역할별 화면의 공통 레이아웃
- 모바일 대응 범위
- 공통 컴포넌트 목록
- 디자인 정리를 언제 후속 작업으로 진행할지
- 실제 화면 구현 후 프로필 사진, 인증 이미지, 평가 결과 시각화의 구체 표현 방식

## 최종 결정

- 참고 레퍼런스: Apple 웹 디자인 참고안
- 디자인 분위기: 깔끔하고 미니멀한 프리미엄 업무 도구 UI
- 디자인 합의 한 줄: 우리 팀 디자인 = Apple식 절제감 + 액션 블루 + 검증된 트레이너 프로필/평가 데이터 중심, 과한 장식과 그림자는 제외
- 메인 컬러: 액션 블루 계열(`#0066cc`)
- 보조 색상: 흰색, 옅은 회색(`#f5f5f7`), 차콜 계열 진한 회색(`#1d1d1f`), 중간 회색
- 폰트: 시스템 폰트
- 제목 기준: 과하게 큰 마케팅형 제목을 피하고, 600 굵기 중심의 조용한 계층 사용
- 본문 크기: 16~17px 기준으로 읽기 쉽게 유지
- 글자 간격: 구현에서는 음수 letter-spacing을 사용하지 않음
- 버튼 기준: 기본, 보조, 위험 버튼으로 구분. 주요 행동 버튼은 파란색 pill 형태, 보조 버튼은 흰색/투명 배경과 얇은 선으로 구분
- 입력창 기준: 라벨, 안내 문구, 오류 문구를 함께 관리
- 카드/표 기준: 카드는 반복 목록과 독립 정보 묶음에 사용하고, 그림자보다 1px 선과 여백으로 구분. 표는 데이터 비교와 관리 화면에 사용
- 역할별 화면 레이아웃: 같은 간격, 카드, 표, 제목 체계를 유지
- 반응형 기준: 모바일에서 텍스트와 버튼이 잘리지 않게 유지
- 이미지 사용 기준: 프로필 사진, 인증 자료, 평가 결과처럼 신뢰를 만드는 실제 정보만 선명하게 사용하고, 흐릿한 장식 배경 이미지는 피함
- 장식 기준: 장식용 그라데이션, 과한 그림자, 페이지별 별도 테마는 사용하지 않음
- 운영 강도: 최소 기준을 맞추고, 약한 디자인 불일치는 후속 정리로 맞춤
- 디자인 정리 기준: 약한 불일치는 후속 정리 가능
- 공통 컴포넌트 운영: 프로젝트 초기에 베이스 공통 컴포넌트 한 벌을 만들어 공유하고, 개발 중 새로 필요한 것은 각자 플로우에서 만듦(중복 감수). 반복되는 패턴은 후속 리팩토링에서 공통으로 통합. 컴포넌트 코드는 라이브로 공유하지 않으며, 라이브 공용 관리는 데이터 이름 사전에만 적용
